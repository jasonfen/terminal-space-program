# v0.5 release notes

The v0.5 series is the **moons + visual enhancement** cycle. v0.5.0
shipped the originally-scoped body-hierarchy work; the v0.5.x patch
line then closed every visual-polish item from the v0.5.3 grab-bag
plus an unplanned but necessary lunar-mission delivery pass.

Latest tag at the time of writing: **v0.5.15**.

---

## v0.5.0 — body hierarchy + 9 major moons

The originally-scoped milestone. Adds arbitrary-depth `parent → child`
body refs so moons can orbit planets correctly.

- `bodies.Body.ParentID` — empty = top-level (orbits system primary);
  set on moons to identify the parent body.
- `BodyPosition` recurses: a moon's inertial position is its parent's
  inertial position plus the moon's parent-relative position.
- `bodyInertialVelocity` recurses identically (parent's μ for the
  velocity component).
- `physics.FindPrimary` uses each body's actual parent for SOI sizing
  (Luna→Earth, Phobos→Mars). Pre-v0.5.0 every body's SOI was sized
  against the system primary, which silently treated moons as planets.
- Moon catalog: **Luna, Phobos, Deimos, Io, Europa, Ganymede,
  Callisto, Titan, Enceladus** — single-moon (Earth) and multi-moon
  (Jupiter, Saturn) primaries both exercised in one release.
- Transfer planning to/from moons explicitly out of scope (slipped
  to v0.6 in the original plan; partially shipped in v0.5.7 — see below).

## v0.5.1 — color palette

`internal/render/palette.go` is the source of truth for body colors,
plus UI-tier colors and stellar-temperature tint. Wired into HUD
selected-body name and body-info screen title.

- Hand-picked colors per body ID (Sol gold-white, Mars rust, Jupiter
  banded ochre, Titan burnt orange, etc).
- `StellarTint(tempK)` for non-Sol stars (M / K / G / F / A / B-O
  buckets).
- UI-tier colors: alert / warning / planted-node / trajectory /
  foreign-SOI / dim — separate so editing one doesn't shift the other.

## v0.5.2 — vessel position trail

200-sample ring buffer of inertial craft positions, sampled every
10 sim seconds (warp-independent density). Renders as a fading-stride
dot trail behind the live Keplerian ellipse.

## v0.5.3 — per-cell canvas body coloring

`Canvas.AddColoredDisk` (replaced in v0.5.10 — see below) tagged each
body's cell-bounding-box with its palette color so the canvas emits
per-cell ANSI foregrounds at `String()` time.

## v0.5.4 — trail follows primary frame

Bug fix: trail samples were stored as heliocentric inertial. For LEO
craft the trail dragged through space at Earth's orbital velocity
(~30 km/s) instead of looping around Earth. Fix: store
`(primaryID, primary-relative R)` per sample; re-translate to current
inertial via `BodyPosition(primary)` at render time. Handles SOI
crossings naturally.

## v0.5.5 — `bodyEphemeris` recurses through hierarchy

Bug fix: `bodyEphemeris` (used by `PorkchopGrid` + `PlanTransferAt`)
didn't recurse through the v0.5.0 hierarchy — moons returned their
parent-relative position as if heliocentric. Earth → Luna porkchop
quoted nonsense Δv. Fix: `bodyHelioStateAt` recurses up the parent
chain.

## v0.5.6 — ICPS-1 default vessel

Pre-v0.5.6 default was 500 kg dry + 500 kg fuel + Isp 300, ~2 km/s
budget. Couldn't reach Luna. Bumped to ICPS-equivalent (Artemis II's
upper stage): 3500 dry + 25000 fuel + Isp 462 + 108 kN, Δv 9.5 km/s.

(Replaced again in v0.5.10 — see below.)

## v0.5.7 — intra-primary Hohmann

Critical bug: `PlanTransfer` always used heliocentric Hohmann math.
For Luna (orbits Earth, same as craft), this interpreted Luna's
parent-relative semimajor as a heliocentric distance — wildly wrong
Δv, ICPS overshoots the moon by orders of magnitude.

Fix: `planner.PlanIntraPrimaryHohmann` for the same-primary case.
`PlanTransfer` dispatches when `target.ParentID == craft.Primary.ID`.
Earth → Luna gives ~3.1 km/s TLI + ~0.7 km/s Luna-orbit insertion —
the textbook patched-conic numbers.

Porkchop screen rejects same-primary targets with a banner directing
the user to the Hohmann auto-plant.

## v0.5.8 — keybind cleanup

Mnemonic swap requested by jason: `P` → porkchop (was `k`),
`H` → Hohmann auto-plant (was `P`). `k` retired.

## v0.5.9 — phase-corrected intra-primary Hohmann

v0.5.7 planted the burn at T+0 with no phase enforcement. Craft
arrived at Luna's altitude but Luna had moved elsewhere. Fix:
`PlanIntraPrimaryHohmann` waits for the next launch window (Luna
leads craft by `π − n_target · T_transfer` at burn moment).
Synodic period for LEO+Luna ≈ 89 min so the wait is short.

Also fixes the porkchop redirect banner from `[P]` to `[H]`.

## v0.5.10 — lunar-mission delivery pass

Bundled fix-chain that took the Earth → Luna mission from "planted
but never reaches" to a working transfer:

- **Tick clamp on burn TriggerTime.** At high warp the tick simDelta
  was 500 s; burns fired up to that late, eating the burn window
  from the wrong side. Tick now clamps simDelta if a finite-burn
  TriggerTime falls within the next tick.
- **Planner pads launch window.** `PlanIntraPrimaryHohmann` adds
  integer synodic periods to wait time until τ ≥ Duration/2, so the
  centered burn doesn't have to fire retroactively.
- **S-IVB-1 default vessel.** Replaced ICPS-1 with the J-2-powered
  Saturn V third stage (1023 kN thrust, Isp 421 s, 11000+40000 kg).
  Δv 6.3 km/s, TLI ≈ 110 s — short enough that finite-burn
  gravity-rotation loss drops to <0.1%, no longer requiring the
  impulsive workaround that v0.5.12-as-then-named introduced.
- **Intra-primary auto-plant returns to finite burns.** Same path as
  inter-primary — finite burn integrated by RK4.
- **TriggerTime semantics flip.** v0.5.14+ ManeuverNode.TriggerTime
  is the burn *center*, not the burn start. Integrator computes
  BurnStart = TriggerTime - Duration/2 internally. HUD T+ display
  shows the planner's intended firing moment, not the
  implementation-detail engine-on time.
- **Per-pixel canvas color tagging.** Replaces v0.5.3's cell-
  rectangle approach. Body color stays confined to body pixels;
  orbit lines / craft glyph / peri-apo markers near a body no longer
  bleed into the body's palette color.
- **Title shows version.** `terminal-space-program — vX.Y.Z — Sol`.

## v0.5.11 — Saturn rings

`render.BodyRings(id)` returns inner / outer ring radii (meters).
Saturn gets the B–A range (92k–137k km from Saturn center). Drawn as
a concentric outline in Saturn's palette color. Extensible for future
ringed bodies.

## v0.5.12 — body-identity glyph overlays

`Canvas.SetCellOverlay` lets the renderer place a Unicode glyph at a
specific cell. `render.GlyphFor(b)` picks one by BodyType +
MeanRadius:

- ☉ star
- ◉ gas giant (radius > 20 000 km)
- ● terrestrial planet
- ○ moon

Body types read distinctly even at small pixel radius — pre-v0.5.12 a
moon and a planet looked identical at system zoom.

## v0.5.13 — HUD section dividers

Replace blank-line section separators with dim `─` rules across the
HUD width. Active-burn block gets a leading ●; peri-below-surface
alert gets a leading ⚠. SYSTEM and SELECTED, previously crammed
together, are now separate sections.

## v0.5.14 — porkchop axis labels

Fix the v0.3.3 misalignment — lead-in width mismatch + label overflow
past grid edge. Tick line `└─────` under the grid, dep-day labels at
every 5th column properly aligned, dim "dep day" axis title.

This closed the last item in the originally-scoped v0.5.3 grab-bag —
all four (rings, glyphs, HUD dividers, porkchop labels) are shipped.

## v0.5.15 — extreme-zoom ring lockup fix

Bug: focusing on a tiny body (Phobos's SOI ≈ 20 m → scale 1.8 px/m)
made Saturn's 137M m outer ring project to ~247M pixels.
`RingColoredOutline` looped `pxRadius * 8` ≈ 2 billion samples and
hard-locked the game.

Two-layer fix:
1. `orbit.go` skips ring draw when `outerPx > canvasReach` (ring
   would be off-canvas anyway).
2. `widgets.RingColoredOutline` and `RingOutline` cap samples at
   4× the canvas pixel diagonal as defense in depth.

## What's deferred to v0.6

Per `docs/state-of-game.md` §3 v0.6:

- **Burn-at-next scheduler** (event-relative maneuver nodes).
- **Mission scaffold** (starter objectives that persist via the
  v0.4.0 save format).
- **Mouse support** (click-to-focus on bodies, click-to-plant on
  porkchop cells, drag-to-edit on planted nodes).
- **Multiplayer design-doc spike** (networking + persistence story).
- **Realistic finite-burn intra-primary auto-plant** (numerical Δv
  iteration to close the remaining ~21% apoapsis over-shoot from
  asymmetric burn termination).
- **Mid-course corrections planning UX** beyond the current scalar
  along-velocity refine.
- **Earth → Luna pathing for `PlanTransfer`** in the inter-SOI
  capture-aware sense (the v0.5.7 path handles the simple
  same-parent case; full multi-SOI patched conics is v0.6).

Plus the architectural `docs/integration-design.md` v0.5.x work —
event-driven outer loop unifying integrator and predictor — which
stayed deferred so the v0.5 series could focus on the originally-
scoped moon + visual content.
